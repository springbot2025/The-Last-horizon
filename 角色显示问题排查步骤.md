# 角色显示问题排查和手动设置步骤

## 问题诊断

如果角色没有显示在地图上，请按以下步骤排查：

## 步骤1：检查场景设置

### 1.1 确保场景中有必要的对象
1. 打开战斗场景（通常是MapScene或包含GridManager的场景）
2. 在Hierarchy中查找：
   - **GameManager** - 必须存在
   - **GridManager** - 必须存在
   - **BattleManager** - 必须存在
   - **Main Camera** - 必须存在

### 1.2 检查GameManager设置
1. 选中场景中的**GameManager**对象
2. 在Inspector中检查：
   - **Grid Manager** 字段必须赋值（拖入场景中的GridManager对象）
   - **UI Manager** 字段可以暂时为空（如果使用CharacterSelectionPanel）

### 1.3 检查GridManager设置
1. 选中场景中的**GridManager**对象
2. 在Inspector中检查：
   - **Width**: 8（默认）
   - **Height**: 6（默认）
   - **Cell Size**: 1（默认）
   - **Tile Prefab** 必须赋值（需要一个GridTile预制体）

### 1.4 检查相机设置
1. 选中**Main Camera**
2. 确认：
   - **Projection**: Orthographic
   - **Size**: 5-10（根据地图大小调整）
   - **Position**: (0, 0, -10) 或类似
   - 确保相机能看到地图范围

## 步骤2：检查动画控制器和精灵

### 2.1 验证动画控制器存在
1. 打开 `Assets/animation` 文件夹
2. 检查以下路径是否有对应的`.controller`文件：
   - `iron wait/images/铁腕待机_01.controller`
   - `gongcheng wait/images/工程师待机_01.controller`
   - `shooter wait/images/弓箭手待机_01.controller`
   - `youxia wait/images/游侠待机_01.controller`
   - `anying wait/images/暗影待机_01.controller`
   - `zhadan wait/images/炸弹兵待机_01.controller`
   - `shengguang wait/images/圣光卫士待机_01.controller`
   - `zhizhong wait/images/至忠圣卫待机_01.controller`

### 2.2 检查动画控制器设置
1. 双击打开任意一个`.controller`文件（如`铁腕待机_01.controller`）
2. 在Animator窗口中检查：
   - 必须有一个默认状态（橙色）
   - 该状态必须连接到一个Animation Clip
   - Animation Clip必须包含sprite帧序列

### 2.3 检查Animation Clip
1. 在Animator窗口中，点击状态节点
2. 在Inspector中查看**Motion**字段
3. 双击Animation Clip，在Animation窗口中检查：
   - **Animation Clip**必须包含**Sprite**关键帧
   - 每一帧应该是一个sprite图片
   - 确保关键帧数量 > 0

## 步骤3：手动测试角色生成

### 3.1 创建测试角色
1. 在Hierarchy中右键 → Create Empty → 命名为 "TestCharacter"
2. 选中TestCharacter，Add Component → 添加`SteelArm`组件（或其他角色组件）
3. Add Component → 添加`CharacterVisualManager`组件
4. Add Component → 添加`Animator`组件
5. Add Component → 添加`SpriteRenderer`组件

### 3.2 手动设置动画
1. 选中TestCharacter
2. 在Inspector中找到**Animator**组件：
   - **Controller**: 拖入`Assets/animation/iron wait/images/铁腕待机_01.controller`
3. 在Inspector中找到**CharacterVisualManager**组件：
   - **Idle Controller**: 拖入同一个controller
4. 在Inspector中找到**SpriteRenderer**组件：
   - **Sorting Layer**: Default（或创建一个新层"Characters"）
   - **Order in Layer**: 10

### 3.3 设置位置
1. 在Inspector中找到**Transform**：
   - **Position**: (0, 0, 0)

### 3.4 运行测试
1. 点击Play按钮
2. 在Scene视图中检查TestCharacter是否显示
3. 如果显示，说明动画控制器正常
4. 如果不显示，检查：
   - Animator的Controller是否正确
   - Animation Clip是否包含sprite
   - SpriteRenderer是否启用

## 步骤4：检查角色预制体（如果需要）

### 4.1 创建角色预制体（推荐）
1. 将TestCharacter从Hierarchy拖到`Assets/Prefabs`文件夹（创建新文件夹）
2. 这会创建一个预制体
3. 删除Hierarchy中的TestCharacter

### 4.2 检查Character脚本初始化
1. 打开`Assets/Character.cs`
2. 确认`Initialize()`方法调用了：
   - `CharacterAnimationLoader.LoadAnimationsForCharacter(this)`
   - `visualManager.SetVisible(true)`
   - `visualManager.PlayIdle()`

## 步骤5：检查控制台日志

### 5.1 运行游戏并查看日志
1. 打开Console窗口（Window → General → Console）
2. 运行游戏，选择角色
3. 查看是否有以下日志：
   - `[CharacterAnimationLoader] Loaded animation controllers for XXX`
   - `[CharacterVisualManager] XXX playing idle animation`
   - `[GameManager] Spawned character XXX with animations at (x, y)`

### 5.2 常见错误信息
- `Animation controller not found` → 动画控制器路径不对
- `missing idleController` → 动画控制器未加载
- `Character spawn failed` → 角色生成失败
- `GridManager is null` → GridManager未设置

## 步骤6：调试方法

### 6.1 添加调试可视化
1. 选中场景中的角色对象（运行时）
2. 在Inspector中检查：
   - **Transform**位置是否正确
   - **Animator**的Controller是否赋值
   - **SpriteRenderer**的enabled是否为true
   - **SpriteRenderer**的sprite是否有值（动画播放时应该有）

### 6.2 检查层级顺序
1. Edit → Project Settings → Tags and Layers
2. 创建新的Sorting Layer: "Characters"（Order = 1）
3. 在角色SpriteRenderer中设置Sorting Layer为"Characters"

### 6.3 临时显示角色
如果动画仍然不显示，可以临时设置一个静态sprite：
1. 在CharacterVisualManager.cs中添加测试代码
2. 或者手动在Inspector中给SpriteRenderer设置一个默认sprite

## 步骤7：手动创建Resources文件夹（如果动画加载失败）

### 7.1 创建Resources文件夹
1. 在Project窗口中，Assets文件夹下右键
2. Create → Folder → 命名为 "Resources"
3. 在Resources下创建 "Animations" 文件夹

### 7.2 复制动画控制器到Resources
1. 将`Assets/animation`下的所有`.controller`文件复制到`Assets/Resources/Animations/`
2. 保持文件名不变

## 快速检查清单

- [ ] 场景中有GameManager对象
- [ ] GameManager的GridManager字段已赋值
- [ ] 场景中有GridManager对象
- [ ] GridManager的Tile Prefab已赋值
- [ ] 场景中有Main Camera
- [ ] 相机设置为Orthographic
- [ ] 动画控制器文件存在于Assets/animation
- [ ] 动画控制器中有Animation Clip
- [ ] Animation Clip包含sprite关键帧
- [ ] 控制台没有错误信息
- [ ] 运行时角色对象的Transform位置正确
- [ ] 运行时角色对象的Animator Controller已赋值
- [ ] 运行时角色对象的SpriteRenderer enabled = true
- [ ] SpriteRenderer的Sorting Order > 0

## 如果以上都检查无误但仍不显示

请提供以下信息：
1. Console中的具体错误信息
2. 运行时的Hierarchy截图
3. 选中角色对象后的Inspector截图
4. Animator窗口的截图
5. Animation窗口的截图（显示Animation Clip内容）
