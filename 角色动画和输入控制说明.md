# 角色动画和输入控制系统说明

## ✅ 已完成的功能

### 1. **角色视觉系统**
- ✅ 创建了 `CharacterVisualManager` 组件，负责管理角色动画
- ✅ 自动添加 `Animator` 和 `SpriteRenderer` 组件
- ✅ 支持四种动画状态：待机（Idle）、移动（Move）、攻击（Attack）、技能（Skill）

### 2. **动画自动加载系统**
- ✅ 创建了 `CharacterAnimationLoader` 静态类
- ✅ 根据角色名称自动查找并加载对应的动画控制器
- ✅ 支持的角色动画映射：
  - 钢腕 → `iron` 系列动画
  - 半魔游侠 → `youxia` 系列动画
  - 弓箭手 → `shooter` 系列动画
  - 暗夜猎手 → `anying` 系列动画
  - 工程师 → `gongcheng` 系列动画
  - 投弹手 → `zhadan` 系列动画
  - 圣光卫士 → `shengguang` 系列动画
  - 至忠圣卫 → `zhizhong` 系列动画

### 3. **键盘输入控制系统**
- ✅ **WASD / 方向键**：控制角色移动（上下左右）
- ✅ **F键**：攻击最近的敌人（在攻击范围内）
- ✅ **J键**：使用主动技能
- ✅ **P键**：使用传送点（如果在传送点上）

### 4. **战斗回合逻辑**
- ✅ 每个回合只能执行一次主操作（移动/攻击/技能）
- ✅ 传送不占用主操作槽位
- ✅ 自动播放对应的动画（移动、攻击、技能）
- ✅ 动画播放完成后自动回到待机状态

## 🎮 操作说明

### 回合开始
当轮到某个角色行动时，Console会显示：
```
等待玩家操作：角色名 (WASD移动, F攻击, J技能, P传送)
```

### 移动操作
- **W / ↑**：向上移动1格
- **S / ↓**：向下移动1格
- **A / ←**：向左移动1格
- **D / →**：向右移动1格
- 移动会播放移动动画，完成后自动回到待机动画

### 攻击操作
- **F键**：攻击攻击范围内最近的敌人
- 如果范围内没有敌人，会提示"攻击范围内没有目标！"
- 如果已执行主操作，会提示"本回合已执行主操作，无法攻击！"
- 攻击会播放攻击动画，并在动画完成后造成伤害

### 技能操作
- **J键**：使用角色的主动技能
- 如果技能已在本局游戏中使用过，会提示无法使用
- 如果已执行主操作，会提示"本回合已执行主操作，无法使用技能！"
- 技能会播放技能动画

### 传送操作
- **P键**：使用传送点
- 只能在站在传送点上时使用
- 传送不占用主操作槽位，可以传送后继续执行其他操作

## 🔧 代码结构

### CharacterVisualManager.cs
负责角色视觉表现：
- `PlayIdle()` - 播放待机动画
- `PlayMove()` - 播放移动动画
- `PlayAttack(Action onComplete)` - 播放攻击动画（带完成回调）
- `PlaySkill(Action onComplete)` - 播放技能动画（带完成回调）
- `FaceDirection(Vector3 target)` - 设置角色朝向
- `SetVisible(bool visible)` - 设置可见性（用于死亡状态）

### CharacterAnimationLoader.cs
负责自动加载动画控制器：
- `LoadAnimationsForCharacter(Character character)` - 为主角加载所有动画
- 支持从Resources或Assets路径加载
- 自动映射角色名称到动画文件

### BattleManager.cs - PlayerTurn方法
处理玩家回合输入：
- 检测WASD移动输入
- 检测F键攻击
- 检测J键技能
- 检测P键传送
- 执行相应的动作并播放动画

### GridManager.cs - MoveCharacter方法
处理角色移动：
- 自动播放移动动画
- 设置角色朝向
- 移动完成后播放待机动画

## ⚠️ 注意事项

### 1. 动画控制器路径
动画控制器需要放在以下位置之一：
- `Resources/Animations/` 文件夹（推荐）
- `Assets/animation/` 文件夹（编辑器模式会自动查找）

### 2. 动画控制器命名规则
动画控制器文件名必须匹配以下模式：
- 待机：`{角色中文名}待机_01.controller`
- 移动：`{角色中文名}移动_01.controller`（特殊：半魔游侠为"行走"，圣光卫士为"移动_06"）
- 攻击：`{角色中文名}攻击_01.controller`
- 技能：`{角色中文名}技能_01.controller`

### 3. 主操作限制
每回合只能执行一次主操作：
- 移动、攻击、技能是互斥的主操作
- 传送是额外操作，不占用主操作槽位
- 执行主操作后，其他主操作将被禁用，直到回合结束

### 4. 攻击目标选择
当前实现会自动攻击攻击范围内最近的敌人。如果需要手动选择目标，可以在UI中添加目标选择功能。

## 🐛 已知问题和限制

1. **动画加载**：如果动画控制器不在正确路径，角色会显示但没有动画
2. **攻击目标**：目前自动选择最近目标，未来可以添加UI选择目标
3. **移动动画**：移动是瞬间的，如果需要平滑移动，需要添加插值动画
4. **传送动画**：传送点使用后没有特殊动画效果
